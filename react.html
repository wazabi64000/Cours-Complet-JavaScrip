1. C’est quoi React ? Et pourquoi l’utiliser ?

React est une bibliothèque JavaScript open source créée par Facebook en 2013 pour construire des interfaces utilisateur dynamiques et interactives.

React ≠ Framework → c’est une bibliothèque. Tu l’utilises quand tu veux, pas comme Angular qui impose tout.

Pourquoi utiliser React ?

  • Réactivité automatique : quand les données changent → l’interface se met à jour seule
  • Composants réutilisables : un bouton, une carte, un menu → tu les crées une fois, tu les réutilises partout
  • Performance : React utilise un DOM virtuel → il ne touche que ce qui change
  • Écosystème énorme : 2 millions de paquets npm, utilisé par Netflix, Airbnb, Uber...
  • Multiplateforme : React Native → même code pour web, iOS, Android

Sans React → HTML + JS classique

<div id="compteur">0</div>
<button onclick="increment()">+1</button>

<script>
let count = 0;
function increment() {
    count++;
    document.getElementById('compteur').innerText = count;
}
</script>

Problème : tu dois tout gérer manuellement → code spaghetti

Avec React → tout est lié automatiquement

function Compteur() {
    const [count, setCount] = useState(0);
    return (
        <>
            <p>{count}</p>
            <button onClick={() => setCount(count + 1)}>+1</button>
        </>
    );
}

Avantage : React gère la mise à jour → tu te concentres sur la logique

2. Premier composant React — Les modules obligatoires

Un composant = un morceau d’interface (bouton, carte, page). Il est réutilisable et indépendant.

Les 3 modules obligatoires (via CDN)

  1. react → le cœur : crée les éléments, gère le virtuel DOM
  2. react-dom → le rendu : insère React dans le navigateur
  3. @babel/standalone → le transpileur : convertit JSX → JS pur
<!-- 1. React (cœur) -->
<script src="https://unpkg.com/react@18/umd/react.development.js"></script>

<!-- 2. ReactDOM (rendu) -->
<script src="https://unpkg.com/react-dom@18/umd/react-dom.development.js"></script>

<!-- 3. Babel (JSX → JS) -->
<script src="https://unpkg.com/@babel/standalone/babel.min.js"></script>
type="text/babel" → indispensable pour que Babel traite le JSX

Premier composant : Bonjour

<div id="root"></div>

<script type="text/babel">
    function Bonjour() {
        return <h1>Bonjour le monde !</h1>;
    }

    const root = ReactDOM.createRoot(document.getElementById('root'));
    root.render(<Bonjour />);
</script>

Ligne par ligne :

  • function Bonjour() → composant fonction (moderne)
  • return <h1>...</h1> → retourne du JSX
  • ReactDOM.createRoot() → prépare le conteneur
  • root.render() → affiche le composant

3. C’est quoi JSX ? Et pourquoi c’est génial ?

JSX = JavaScript XML. C’est une syntaxe qui permet d’écrire du HTML-like dans du JavaScript.

Ce n’est PAS du HTML ! C’est du sucre syntaxique → Babel le transforme en appels React.createElement()

Exemple avant/après

// JSX (ce que tu écris)
<div className="card">
    <h2>Titre</h2>
    <p>Contenu</p>
</div>

// ↓ Babel transforme en ↓

React.createElement("div", { className: "card" },
    React.createElement("h2", null, "Titre"),
    React.createElement("p", null, "Contenu")
);

Règles JSX importantes

  • class → className (car class est un mot réservé JS)
  • for → htmlFor
  • Toutes les balises doivent être fermées : <br />
  • Un seul élément racine → utilise <></> (Fragment)
  • Expressions JS dans {} : <p>2 + 2 = {2+2}</p>
Pourquoi JSX ? Parce que UI = données + logique. Mettre les deux au même endroit = plus clair, plus maintenable.

4. Deux exemples clairs : Classe vs Fonction

Exemple 1 : Compteur avec Classe (2013–2018)

class CompteurClasse extends React.Component {
    constructor(props) {
        super(props);
        this.state = { count: 0 };
        this.increment = this.increment.bind(this); // Obligatoire !
    }

    increment() {
        this.setState({ count: this.state.count + 1 });
    }

    render() {
        return (
            <div>
                <p>Compte : {this.state.count}</p>
                <button onClick={this.increment}>+1</button>
            </div>
        );
    }
}

Points clés : this.state, this.setState(), bind() dans le constructeur

Exemple 2 : Même compteur avec Hooks (depuis 2019)

function CompteurFonction() {
    const [count, setCount] = React.useState(0);

    return (
        <div>
            <p>Compte : {count}</p>
            <button onClick={() => setCount(count + 1)}>+1</button>
        </div>
    );
}

Avantages : Moins de code, pas de this, plus lisible, réutilisable

En 2025 → 99 % des nouveaux projets utilisent les hooks.
Mais comprendre les classes = comprendre l’évolution et les projets legacy.

5. Exemple moderne : Todo List avec hooks

Todo App complète (useState + useEffect)

<script type="text/babel">
function TodoApp() {
    const [todos, setTodos] = React.useState([]);
    const [input, setInput] = React.useState('');

    // Charge les todos depuis localStorage
    React.useEffect(() => {
        const saved = localStorage.getItem('todos');
        if (saved) setTodos(JSON.parse(saved));
    }, []);

    // Sauvegarde à chaque changement
    React.useEffect(() => {
        localStorage.setItem('todos', JSON.stringify(todos));
    }, [todos]);

    const ajouter = () => {
        if (input.trim()) {
            setTodos([...todos, { id: Date.now(), texte: input, fait: false }]);
            setInput('');
        }
    };

    const toggle = (id) => {
        setTodos(todos.map(t => 
            t.id === id ? { ...t, fait: !t.fait } : t
        ));
    };

    const supprimer = (id) => {
        setTodos(todos.filter(t => t.id !== id));
    };

    return (
        <div style={{maxWidth: 500, margin: 'auto', padding: 20}}>
            <h1>Ma Todo List</h1>
            <div>
                <input 
                    value={input} 
                    onChange={(e) => setInput(e.target.value)}
                    onKeyPress={(e) => e.key === 'Enter' && ajouter()}
                    placeholder="Nouvelle tâche"
                />
                <button onClick={ajouter}>Ajouter</button>
            </div>
            <ul style={{listStyle: 'none', padding: 0}}>
                {todos.map(todo => (
                    <li key={todo.id} style={{
                        display: 'flex', justifyContent: 'space-between',
                        padding: '10px', margin: '5px 0',
                        background: todo.fait ? '#e0f7fa' : '#fff',
                        textDecoration: todo.fait ? 'line-through' : 'none'
                    }}>
                        <span onClick={() => toggle(todo.id)} style={{cursor: 'pointer'}}>
                            {todo.texte}
                        </span>
                        <button onClick={() => supprimer(todo.id)} style={{color: 'red'}}>✕</button>
                    </li>
                ))}
            </ul>
        </div>
    );
}

const root = ReactDOM.createRoot(document.getElementById('root'));
root.render(<TodoApp />);
</script>

Fonctionnalités : Ajouter, cocher, supprimer, persistance dans localStorage
